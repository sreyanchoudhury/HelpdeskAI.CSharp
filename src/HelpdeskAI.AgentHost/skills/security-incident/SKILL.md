# Security Incident Response

Use this skill when a user reports anything that may indicate a security breach, compromise, or threat. Examples: received a suspicious email, clicked a phishing link, noticed unexpected account activity, device acting strangely, received a ransom message, or discovered data may have been exposed.

## Critical Rule

**Never attempt to diagnose or resolve a security incident through standard IT troubleshooting alone.** Always involve the Security team. Time matters — a delayed response can significantly expand the impact of a breach.

## Immediate Triage

Ask these questions quickly:
1. What did they see or click?
2. When did it happen (approximate time)?
3. What device and accounts are involved?
4. Have they shared any credentials or sensitive information?
5. Are they still seeing the behaviour (active threat vs. past event)?

## Response Actions

**Step 1 — Contain**
- If credentials may be compromised: advise the user to change their password immediately from a different, trusted device
- If a device may be infected: advise them to disconnect from the corporate network immediately (unplug ethernet / disable Wi-Fi)
- Do NOT advise them to restart — this may clear forensic evidence

**Step 2 — Report**
- Tell the user: "Please call the Security Hotline: ext. 9911" — this is mandatory for all security incidents
- Create a ticket with priority "Critical" and category "Security"
- Add all collected details (what happened, device, time, accounts) to the ticket comment

**Step 3 — Preserve Evidence**
- Advise the user not to delete suspicious emails — forward them to security@contoso.com
- Do not run antivirus scans without Security team guidance — some scans can destroy forensic artefacts

**Step 4 — Communicate**
- Confirm to the user that the Security team has been notified and will contact them
- Give the ticket ID as a reference
- Reassure without minimising: "You did the right thing by reporting this immediately"

## What NOT to Do

- Do not access the suspicious link or attachment yourself to investigate
- Do not promise that "nothing bad will happen" — you cannot know that yet
- Do not delay escalation to try to resolve it yourself first
- Do not document specific breach details in public-facing chat logs — use the ticket system

## Tone

- Calm and clear — avoid causing panic, but communicate the seriousness
- Avoid: "Oh no, that's really bad" — stay measured
- Use: "I'm treating this as a priority and the Security team will take it from here"
- Acknowledge that the user reporting it was the right action — this encourages future reporting culture
